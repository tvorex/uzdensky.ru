Общее положение

Сайт создается на CMS Joomla 3.x под ключ клиенту.
Задача: сверстать адаптийный шаблон под все интерективные устройва и с единой панелью управления. С возможностью добавления материалов, их корректировки для коректно отображался на всех устройствах.

Левая часть шаблона всегда статична и имеет ширину не менее 370px. Правая часть является резиновой и растягивается на оставшеюся ширину сайта

Макет под устройства "Tablets" так же имеет статичную левую часть и логотип, все остально растягивается на всю ширину сайта

Логотип

Список может меняться, как добавляться так и исключаться...

Меню

Имеется два вида меню: Системное и Метки. Ось вращения логотипа горизонтальная, переворот сверху вниз, с периодичностью 7 секунды. При наведении курсором = Паузы нет

Системное меню

Дизайн утвердили, но выравнивание еще неясно. Возможно равнение будет относительно текста, возможно относительна бегунков. Текст может двигаться вверх или вниз. Это меню по разделам сайта. Обратите внимание на слэш. Он является эементом дизайна и не относится к тексту. На разных устройствах он будет отображаться по разному, а в footer его вовсе нет. В Footer возможно понадобиться заменить цвет фона.

Меню Метки или Поиск текста под настроение

По сути, это список меток, тэгов. К каждому материалу у клиента должны быть возможность добавлять несколько меток из заранее подготовленного списка. При нажатие на одну из меток, должен выводиться список материалов (не поиск), в которых встречается данная метка. Без использования функции "Поиск по меткам". Рекомендую использовать RokSprocket. При наведении курсором мыши на стрелки, список должен пролистоваться в соответствующую сторону. При наведении курсора мыши на название (саму метку), красный ползунок должен двигаться к нему.

На мобильных устройствах это меню приобретает дополнительные иконки. Возможно, если будет готовые список меток, к каждой из которых мы сможем указать рисунок.

Фото и цитаты

С каждым обновлением или открытием новой страница появляется случайна цитата и случайное фото. Цитата - это пару строк из полного стихотворения (материала), при нажатие на иконку раскрытие открывается выпадающее окно на весь экран подобно полному материалу и пользователь читает в нем полное стихотворение.

Цитати в мобильной и small версиях тоже будут крутиться как и лого, сверху вниз. Тем самым менять текст и фото

    Контент материала - это стихи автора. Главное, ради чего создаетя сайт. Контентная часть должна быть полностью редактируемой html разметкой в административный панели. Элемент "Книга" будет только на страницах материалов. Рекоменду использовать RokPad

    1. Стрелки влево - ведут на ранне опубликованый материал в текущей категории, если материал. Стрелки вправо - ведут на следующий материал в данной категории. Если это последний материал, то открывается первый материал в данной категории
    2. Это изображение материала. Вот с ним сложно.
      • Если у нас книга издана, то картинка будет статичная, но разного формата для различных устройств. Клиент будет готовить 4 картинки формата name_large name_medium name_small name_small_extra. Так же у нас будет повторяющая картинка для общего фона, в данном случае бежевого цвета
      • Если книга еще не вышла, у стихотворения будет стандартный повторяющийся рисунок (так же 4 вида, для разных устройств) и рамка по центру (цвет которой выбирается индивидуально для каждого материала), внутри которой стихотворение. Возможно это будет делать через стиль, возможно с использование готовой палитры цветов, что приветствуется больше. Повторяющийся рисунок выбирается в панеле управления материалов, но если он не выбран используется картнка для категорий
      • Обратите внимание, что отспуты у нас могут быть разные и должны настраиваться в ручную для каждого стихотворения. Для этого и требуется сохранить html разметку в административной панели. В целяю безопасности, продумать шаблон так, что бы ошибки в верстке не разрушили общий вид сайта.
      • Как я это вижу: Мы создаем материал и ставим галочку, вышел материал в печать или нет. В зависимости от выбора подключается html файл для контента с различными условиями. Если text_published то у нас будет два поля для выбора картинки материала и повторяющейся картинки, так же используется нестандартный шрифт, который нужно будет подключать именно в материале, для каждого стихотворения уникальный шрифт. Если text_not_published, то мы используем стандартную функцию изображения полного материала, а сам материал отображается в оранжевой рамке... Как-то так.
    3. Заголовок и основной текст. Некоторые строчки могут иметь отступ, это делается в ручную с использование CSS. Так же в стихотворениях возможно визуальное выделение слов. Для этих целей требуется создать отдельный файл style_content.css который будет со временем доплняться различными стилями для оформления стихотворений.

Если данного файла нет, то иконка не выводится. При переходе к другому материалу аудио файл автоматически отключается.
      • Открывает в новом окне версию для печати или сразу отправляет страницу на печать.
      • Раскрывает стихотворение на весь экран, по сути это таже самая версия для печати, но только открываетя при помощи всплывающего окна по верх всего контента
      • Всплывающее окно, убирает все спец эффекты с текста.

    Листинг книг или "Поиск по книгам"

    Модуль отображает книги (категории) автора. Книги медленно двигаются снизу вверх, при наведении курсором на большую стрелку скорость прокручивания увеличивается. Если навести на область книги или иконки, то прокручивание прекращается = Пауза.

    Каждая книга (раздел) имеет страницу описания. Кликнув на книгу, мы попадаем на эту страницу. Данные иконки - это анкоры той же страницы.

    Обложка - это просто картинка, будет готовится клиентом в программе Photoshop.

Функция "Удалить комментари" только Супер-пользоветлю. Авторизация производится через административную панель. Серия (дерево) комментариев разделены горизонтальной линией

    Вопрос: Сейчас делаем ответ под комментарием, возможно в будущем изменить, что ответ отображался как новый комментарий?

    Страница предназначена для публикации отзывов пользователей и введение дискуссий (Вопросы автору). Страница снабжена функциями:

    • Администратор имеет дополнительную функцию "Удалить" ответ
